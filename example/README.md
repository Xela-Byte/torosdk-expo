# ToroSDK Expo Example

This example demonstrates the complete [torosdk-expo](https://www.npmjs.com/package/torosdk-expo) SDK — a plug-and-play React Native wrapper around the Toronet blockchain.

It uses the **published npm package** (`torosdk-expo ^0.1.1`), the same way any consumer would after running `npx torosdk-expo init`.

## Quick Start

```bash
cd example
npm install --legacy-peer-deps
npx expo start
```

Then press `i` for iOS simulator or `a` for Android emulator.

> **Note:** `--legacy-peer-deps` is needed because `react-native-screens@4.x` requires `react-native >= 0.82` but Expo SDK 52 pins `react-native 0.76.9`. The app works fine — this is a cosmetic peer range mismatch.

## What's Inside

| Screen | Hooks Demonstrated |
|--------|-------------------|
| **Home** | `useWallets`, `useBalances` |
| **Wallets** | `useCreateWallet`, `useImportWallet`, `useDeleteWallet`, `useWallets` |
| **Send** | `useTransfer`, `useBalance`, `useWallets`, `Currency` |
| **TNS Names** | `useResolveTNS`, `useLookupTNS`, `useSetTNS`, `useWallets` |
| **KYC Status** | `useKYCStatus`, `useWallets` |
| **Exchange Rates** | `useExchangeRates` |

## How It Was Created

This example was bootstrapped to match exactly what a consumer does:

1. `npx create-expo-app@latest example --template blank-typescript@52.0.0`
2. `npx expo install @react-navigation/native @react-navigation/native-stack react-native-screens react-native-safe-area-context`
3. `npx torosdk-expo init` — scaffolds `src/torosdk/` and installs SDK dependencies
4. Custom screens added to `src/screens/`
5. `App.tsx` wired up with navigation + `ToroWrapper`
6. `app.json` configured with iOS ATS exception (testnet HTTP), FaceID usage description, and Expo plugins

## Differences from Old Example

The old example resolved `torosdk-expo` from local source via Metro `extraNodeModules` and `tsconfig.json` `paths`. This version uses the published npm package — no Metro customization, no path aliases, no need to build the parent repo first.
